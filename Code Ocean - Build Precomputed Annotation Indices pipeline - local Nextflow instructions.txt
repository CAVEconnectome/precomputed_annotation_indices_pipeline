____________________________________________________________________________________________________
NEXTFLOW AND DOCKER PREREQUISITES
----------

First, install Nextflow, which itself will require installing Java:
  https://www.nextflow.io/docs/latest/install.html

Then, on a Mac at any rate, you need to download and run the Docker desktop app. Go for it. I can't offer guidance on other platforms, but I'm sure it's similar. Nextflow will then utilize the running Docker environment for the pipeline.

____________________________________________________________________________________________________
DOWNLOADING THE PIPELINE
----------

Clone the repo:
$ git clone https://github.com/CAVEconnectome/precomputed_annotation_indices_pipeline.git
$ cd precomputed_annotation_indices_pipeline

You will get a directory that contains the follow (plus some other stuff):
precomputed_annotation_indices_pipeline/
  metadata/
    metadata.yml
  pipeline/
    main.nf
    nextflow.config
  data_config_template.json  data_config_example.json
  data/
    <...3 subdirectories...>

Theoretically, you can run this repo right out of the box since it includes a small sample of data. So if you want to skip to the bottom to see if it'll work, give it a shot. Once you confirm that you have Nextflow and Docker all set up correctly, return to this point and go through these instructions to configure a run against your own data. 

____________________________________________________________________________________________________
DATA
----------

You need to move your data into the repo at the path where Nextflow will expect to find it. The repo includes a small example to show you the necessary layout. The pipeline will expect to find the data in a data/ directory at the same level as the pipeline/ directory (so, outside, but next to, the pipeline/ and metadata/ directories). Within data/, there should be at least two subdirectories, one to hold the data-config file and one to hold the data. There can be additional data directories, which can be used to indicate smaller test subsets. Here is an example showing one such subset named "__test":

pipeline/
  ...
metadata/
  ...
data/  synapses_pni_2_v1_filtered_view__v1412__data-config/    synapses_pni_2_v1_filtered_view__v1412__data-config.json  synapses_pni_2_v1_filtered_view__v1412__test/
    synapses_pni_2_v1_filtered_view__1-root-ids_337312429-row-limit_2108-rows.csv  synapses_pni_2_v1_filtered_view__v1412/
    synapses_pni_2_v1_filtered_view__10-root-ids_337312429-row-limit_24426-rows.csv

Or without a test subset, it would look like this:

data/  synapses_pni_2_v1_filtered_view__v1412__data-config/    synapses_pni_2_v1_filtered_view__v1412__data-config.json  synapses_pni_2_v1_filtered_view__v1412/
    synapses_pni_2_v1_filtered_view__10-root-ids_337312429-row-limit_24426-rows.csv
____________________________________________________________________________________________________
CREATING THE DATA CONFIG FILE
----------

You already have your data; you brought that on your own. But you need to create the data-config.json file as well, indicated above. Creating such a file is described in detail in the Git repo README (which will have come down to your local machine when you cloned it at the top of these instructions; it's next to this instructions.txt file in fact), so I won't repeat the same information here, but note the relationship between the "data_sizes" section of that json file and the various directory names shown in the previous section. Make sure they align. There should be a data_size entry for the overall data and for any subset you wish to create. For the data layout shown above, the data-config file would need a "data_sizes" section that looks like this:

    "data_sizes" : {
        "synapses_pni_2_v1_filtered_view__v1412__test":  [313245,  2108],
        "synapses_pni_2_v1_filtered_view__v1412":       [3645906, 24426]
    },

Data subsets are optional. Without one, the "data_sizes" section would simply look like this:

    "data_sizes" : {
        "synapses_pni_2_v1_filtered_view__v1412":       [3645906, 24426],
    },

Obviously, you can name your data anything want so long as you keep track of using the same string in the necessary places.

____________________________________________________________________________________________________
RUNNING THE PIPELINE
----------

When running the pipeline, you specify various Nextflow configuration files as well as the overall data name and the specific data subset you wish to run. If you only have one (the entire dataset) or you intend to process the entire dataset in a give run, then pass in the same string for both the "data_name" and "datasubset_name" arguments below. To run a subset, leave the data_name the same but alter the datasubset_name.

You need to be in the pipeline/ directory to run it, so:
$ cd pipeline

To run the pipeline against a subset, do this (substituting your own data and subset names in the last two lines of the command):
$ rm -rf work && \
  DATA_PATH=$PWD/../data \
  nextflow -c nextflow_all_indices.config -log ../results/nextflow/nextflow.log \
  run main_noncodeocean_all_indices.nf \
  --data_name       synapses_pni_2_v1_filtered_view__v1412 \
  --datasubset_name synapses_pni_2_v1_filtered_view__v1412__test

Or to process a full dataset, indicate the full dataset (by simply passing the same value for both arguments):
  --data_name       synapses_pni_2_v1_filtered_view__v1412 \
  --datasubset_name synapses_pni_2_v1_filtered_view__v1412

Depending on the development state of the repo, you may see some extraneous files, such as the following:
  nextflow_id_idx.config (only builds the ID index)  main_noncodeocean_id_idx.nf (only builds the ID index)
  nextflow_id_relation_indices.config (only builds the ID and relation indices)  main_noncodeocean_id_relation_indices.nf (only builds the ID and relation indices)  main.nf (leftover)
You can safely dismiss them. I intend to clean things up, but the repo remains in a state of ongoing development.  
____________________________________________________________________________________________________
SEEING THE RESULTS
----------

After running, you will see a results/ directory. Note that none of the directories are files are direct file references however. They are all links to results files that actually reside inside individual capsule work directories in the pipeline/work/ directory.

____________________________________________________________________________________________________
WORKING WITH DISPARATE DATA
----------

Notice how easy it is to work with numerous, unrelated datasets. You may be working on synpase data and axon data, or on fly data and mouse data, or multiple different data across a wide variety of domains and projects. Simply give each dataset a unique name and collect them all in the data/ directory, along with their associated data-config.json file, which you must create once for each new dataset. Then you can indicate whichever dataset (and test subset) you want with the run command.
